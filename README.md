# 📊 Uber-Ride-Analysis-SQL

## 📌 Project Description
Analyzed Uber ride data to identify patterns in bookings, demand, and ride trends. Used SQL queries to explore peak hours, popular locations, and trip frequencies to gain insights into user behavior and ride performance.



---

## 🎯 Objectives
-To analyze Uber ride data and understand booking patterns and trends
-To identify peak hours, high-demand locations, and ride frequency
-To study trip distribution based on time, location, and demand
-To extract insights using SQL queries such as filtering, grouping, and aggregation

---

## 🛠️ Tools & Technologies
- SQL (MySQL)  
- MySQL Workbench / DBeaver (Database Tool)  
- GitHub  

---

## 📂 Dataset Description
- Database Name: ft_a 
- Tables Name:
- drivers
- passangers
- rides

---


## 📈 Key Insights  
- Identifies ride patterns, demand, and popular pickup locations  
- Shows overall rides, earnings, and average fare trends  
- Highlights driver performance and top-rated drivers  
- Analyzes customer behavior through payment methods and ride distance
